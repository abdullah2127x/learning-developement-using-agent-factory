# Setup & Configuration

## Installation

```bash
npm install next-auth@beta
npx auth secret   # Generates AUTH_SECRET in .env.local
```

**Requirements**: Next.js 14.0+ (App Router), TypeScript 5.0+

## auth.ts — Central Configuration

```typescript
// auth.ts (project root)
import NextAuth from "next-auth";
import GitHub from "next-auth/providers/github";
import Google from "next-auth/providers/google";

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [GitHub, Google],
});
```

**Exports**:
- `handlers` — `{ GET, POST }` for route handler
- `signIn` — Server-side sign in function
- `signOut` — Server-side sign out function
- `auth` — Universal session getter (Server Components, Route Handlers, middleware)

## Route Handler

```typescript
// app/api/auth/[...nextauth]/route.ts
import { handlers } from "@/auth";
export const { GET, POST } = handlers;
```

This creates all auth API routes automatically:
- `GET /api/auth/signin` — Sign-in page
- `POST /api/auth/signin/:provider` — Start OAuth flow
- `GET /api/auth/callback/:provider` — OAuth callback
- `POST /api/auth/signout` — Sign out
- `GET /api/auth/session` — Get session JSON
- `GET /api/auth/csrf` — CSRF token
- `GET /api/auth/providers` — List providers

## Environment Variables

### Required

```env
AUTH_SECRET="your-secret-here"   # Generated by `npx auth secret`
```

### OAuth Provider Convention

Auth.js v5 auto-detects environment variables following this pattern:

```env
AUTH_{PROVIDER}_ID="client-id"
AUTH_{PROVIDER}_SECRET="client-secret"
```

Examples:
```env
# GitHub
AUTH_GITHUB_ID="abc123"
AUTH_GITHUB_SECRET="xyz789"

# Google
AUTH_GOOGLE_ID="abc123.apps.googleusercontent.com"
AUTH_GOOGLE_SECRET="xyz789"

# Discord
AUTH_DISCORD_ID="abc123"
AUTH_DISCORD_SECRET="xyz789"
```

When using auto-detection, no config needed in provider:
```typescript
import GitHub from "next-auth/providers/github";
// Just: GitHub  (no { clientId, clientSecret } needed)
```

### Manual Environment Variables

If you use custom env var names:
```typescript
import GitHub from "next-auth/providers/github";

export const { handlers, auth } = NextAuth({
  providers: [
    GitHub({
      clientId: process.env.CUSTOM_GITHUB_ID,
      clientSecret: process.env.CUSTOM_GITHUB_SECRET,
    }),
  ],
});
```

### trustHost

In production behind a proxy, set:
```typescript
export const { handlers, auth } = NextAuth({
  trustHost: true,  // Trust X-Forwarded-Host header
  providers: [...],
});
```

Or via environment: `AUTH_TRUST_HOST=true`

**Note**: `NEXTAUTH_URL` is no longer needed in v5 — auto-detected.

## Full Configuration Object

```typescript
import NextAuth from "next-auth";
import type { NextAuthConfig } from "next-auth";

const config: NextAuthConfig = {
  // Providers
  providers: [],

  // Callbacks (jwt, session, signIn, redirect, authorized)
  callbacks: {},

  // Session strategy
  session: {
    strategy: "jwt",        // "jwt" (default) or "database"
    maxAge: 30 * 24 * 60 * 60,  // 30 days
    updateAge: 24 * 60 * 60,    // 24 hours
  },

  // Custom pages
  pages: {
    signIn: "/auth/signin",
    signOut: "/auth/signout",
    error: "/auth/error",
    verifyRequest: "/auth/verify",
    newUser: "/auth/new-user",
  },

  // Database adapter (optional)
  // adapter: PrismaAdapter(prisma),

  // JWT options
  jwt: {
    maxAge: 30 * 24 * 60 * 60,  // 30 days
  },

  // Events (logging, side effects)
  events: {
    signIn: async ({ user, account, profile }) => {},
    signOut: async ({ session, token }) => {},
    createUser: async ({ user }) => {},
    linkAccount: async ({ user, account, profile }) => {},
    session: async ({ session, token }) => {},
  },

  // Debug mode
  debug: process.env.NODE_ENV === "development",

  // Trust host
  trustHost: true,
};

export const { handlers, signIn, signOut, auth } = NextAuth(config);
```

## Edge-Safe Config Split

When using a database adapter AND middleware, split the config:

### auth.config.ts (Edge-compatible — no adapter)

```typescript
// auth.config.ts
import GitHub from "next-auth/providers/github";
import type { NextAuthConfig } from "next-auth";

export default {
  providers: [GitHub],
  pages: {
    signIn: "/auth/signin",
  },
  callbacks: {
    authorized: ({ auth, request: { nextUrl } }) => {
      const isLoggedIn = !!auth?.user;
      const isProtected = nextUrl.pathname.startsWith("/dashboard");
      if (isProtected && !isLoggedIn) return false;
      return true;
    },
  },
} satisfies NextAuthConfig;
```

### auth.ts (Full config with adapter — NOT edge-safe)

```typescript
// auth.ts
import NextAuth from "next-auth";
import { PrismaAdapter } from "@auth/prisma-adapter";
import { prisma } from "@/lib/prisma";
import authConfig from "./auth.config";

export const { handlers, signIn, signOut, auth } = NextAuth({
  ...authConfig,
  adapter: PrismaAdapter(prisma),
  session: { strategy: "database" },
  callbacks: {
    ...authConfig.callbacks,
    session: ({ session, user }) => ({
      ...session,
      user: { ...session.user, id: user.id },
    }),
  },
});
```

### middleware.ts (uses edge-safe config)

```typescript
// middleware.ts
import NextAuth from "next-auth";
import authConfig from "./auth.config";

export const { auth: middleware } = NextAuth(authConfig);

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```

## Using auth() Everywhere

### Server Component

```typescript
import { auth } from "@/auth";

export default async function Page() {
  const session = await auth();
  if (!session?.user) return <div>Not authenticated</div>;
  return <div>Hello {session.user.name}</div>;
}
```

### Server Action

```typescript
"use server";
import { auth } from "@/auth";

export async function updateProfile(data: FormData) {
  const session = await auth();
  if (!session?.user) throw new Error("Unauthorized");
  // ... update profile
}
```

### Route Handler

```typescript
import { auth } from "@/auth";
import { NextResponse } from "next/server";

export const GET = auth(function GET(req) {
  if (!req.auth) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }
  return NextResponse.json({ user: req.auth.user });
});
```

### Client Component (via SessionProvider)

```typescript
// app/layout.tsx
import { SessionProvider } from "next-auth/react";

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <body>
        <SessionProvider>{children}</SessionProvider>
      </body>
    </html>
  );
}
```

```typescript
// components/user-menu.tsx
"use client";
import { useSession, signIn, signOut } from "next-auth/react";

export function UserMenu() {
  const { data: session, status } = useSession();

  if (status === "loading") return <div>Loading...</div>;
  if (!session) return <button onClick={() => signIn()}>Sign in</button>;

  return (
    <div>
      <p>{session.user?.name}</p>
      <button onClick={() => signOut()}>Sign out</button>
    </div>
  );
}
```

## Server Actions for Sign In/Out

Preferred over client-side calls:

```typescript
// components/sign-in.tsx (Server Component)
import { signIn } from "@/auth";

export function SignIn() {
  return (
    <form
      action={async () => {
        "use server";
        await signIn("github");
      }}
    >
      <button type="submit">Sign in with GitHub</button>
    </form>
  );
}
```

```typescript
// components/sign-out.tsx
import { signOut } from "@/auth";

export function SignOut() {
  return (
    <form
      action={async () => {
        "use server";
        await signOut();
      }}
    >
      <button type="submit">Sign out</button>
    </form>
  );
}
```
